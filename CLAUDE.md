# Borderless — 교회 청년부 수련회 미션 앱

## 상태

참가자 플로우(팀/조 선택 → 지도 → 사진 제출 → AI 판정 → 미션 공개 → 영상 업로드 → 인벤토리)와 임원 플로우(비밀 경로 setup CRUD, 도움 요청, 지급 확정, 대시보드, 영상 모아보기) 모두 구현 완료, Vercel에 배포됨 (2026-07-15). 이후 실기기 테스트를 반복하며 발견된 버그들(카메라 강제 연동, 사진 크롭, 지도 마커 상태 미갱신, 패널 상태 누수, 확대 오버레이가 실기기에서 안 닫힘 등)을 계속 수정했고, 어드민 쪽에 사진/미션/재료/판정질문 사후 수정, 제출 초기화, 팀 선택 잠금 기능도 추가함 (2026-07-19). 2026-07-20에는 어드민 "새 포인트 추가" 폼에 지도 클릭 좌표 지정(주소 검색 시 자동 확대·이동)을 넣었고, 미션 인라인 수정, 포인트를 삭제 없이 숨길 수 있는 활성/비활성 토글, 나침반 권한을 기억해 다음 방문부터 재요청 안 하게 하는 것, 새 포인트 등록 후 폼 자동 초기화를 추가했다. 2026-07-21에는 Ingredient 모델에서 실제로 안 쓰이던 category/variant/isBase 필드를 정리하고 미션에 퀴즈(PUZZLE) 타입을 추가했으며(영상 업로드 대신 정답 텍스트 제출로 완료), 도움 요청에 조원 이름과 메시지가 함께 남게 했고, 참가자가 매번 입력하던 조원 이름을 어드민이 미리 등록해두는 방식으로 전환했다(팀 선택 화면에서 이름 입력 제거). 지도 마커에 선택 상태·비활성(차례 아님/마감) 표시를 다듬었고, 미션 수행 영상 안내를 "10~20초"에서 "30초 이상"으로 조정했다. 2026-07-22에는 어드민 화면을 **설정**(지역/미션/재료/방문포인트)과 **팀**(팀·조 이름/조원, 그룹 선택 잠금, 그룹별 지역 방문 순서, 그룹별 진행, 도움 요청, 포인트별 현황·지급확정) 2탭으로 재편했고(기존 도움요청·대시보드 페이지 통합), 지도에 "같은 팀 다른 조가 이 지역을 이미 통과했다"는 안내를 추가했다. PC 화면에서는 지도(왼쪽)·상세 패널(오른쪽) 6:4 비율 2단 레이아웃을 별도로 지원한다(모바일은 기존 그대로 세로 스택). 차례가 아닌 지역/이미 마감된 포인트를 선택했을 때는 평범한 안내 문장 대신 PASS 카드와 같은 골격의 LOCKED/CLOSED 카드로 강조 표시한다. 남은 건 답사(현장 답사 후 실제 16개 포인트 등록·프롬프트 조정)와 그룹별 지역 방문 순서 확정(다른 임원 소관, 미확정) — 이 둘 다 이 리포지토리 밖에서 정해져야 하는 부분. 진행 상황·이슈는 [notes/](notes/) 디렉터리에 날짜별로 기록. 상세 기획은 [Borderless_클로드코드_개발기획서.md](Borderless_클로드코드_개발기획서.md)가 원본(source of truth)이니, 세부 사항은 항상 그 문서를 먼저 확인할 것 — 아래는 빠르게 감을 잡기 위한 요약일 뿐 이 파일을 근거로 세부 구현 결정을 내리지 말 것.

이 프로젝트는 Next.js 템플릿이 생성한 `AGENTS.md`도 포함하고 있음 — 이 Next.js 버전이 훈련 데이터와 다를 수 있다는 경고이니 코드 작성 전 참고할 것.

## 한 줄 요약

참가자(8그룹)가 지도로 안내된 야외 지점에서 사진을 찍어 올리면 AI(VQA 방식)가 기준 사진과 비교해 통과 여부를 판정하고, 통과 시 미션(말씀/찬양/기도/퀴즈)이 공개되며 재료를 모아가는 모바일 웹앱.

## 핵심 구조 (변경 시 기획서부터 갱신)

- 4팀(A~D) × 팀당 2조 = 8그룹. 4지역(a~d) × 지역당 4포인트 = 총 16포인트.
- 그룹은 지역마다 포인트 1곳씩, 정확히 4곳만 방문 (그 이상도 이하도 없음).
- **그룹별로 지역 방문 순서가 강제됨** (다른 임원이 확정 예정, 개발 중엔 더미 순서). 지도 마커 자체는 잠그지 않지만, 차례가 아닌 지역의 장소 상세(기준 사진/업로드)는 숨기고 대신 LOCKED 카드로 안내(같은 지역이라도 미션 영상 업로드가 안 끝났으면 CLOSED가 아니라 별도 안내). 이미 마감된 포인트는 CLOSED 카드. 서버에서도 이중 검증.
- 포인트당 선착순 2그룹 캡. **캡 확인은 AI 판정 전에 먼저 실행되고, 참가자에게는 방문 전 절대 노출 안 됨** — 사진 업로드 시점에 처음 "마감" 여부를 알게 됨. AI 통과 즉시 서버가 캡을 원자적으로 차감(동시성 처리 필수). 마감 확인 정보는 **팀 단위**로 공개(같은 팀 다른 조에게도 보임, 타 팀은 비공개).
- 재료의 물리적 지급은 오프라인(임원이 현장에서 직접). 관리자 화면의 "지급 확정"은 캡을 가르는 게이트가 아니라 전달 기록용. 처리자가 누구인지는 기록하지 않고 완료 여부만 추적(임원이 소수라 불필요).
- 재료 교환 기능은 이번 버전 범위에서 완전히 제외 (오프라인으로 처리).
- **지역 "완료"는 AI 판정 통과만으론 안 되고, 미션 완료(대부분 영상 업로드, 퀴즈(PUZZLE) 타입은 예외로 정답 텍스트 제출)까지 끝나야 확정된다** (2026-07-15 확정, 2026-07-21 퀴즈 타입 추가). 그 전까지는 상단 진행 표시·"다음 목적지" 안내가 그 지역에 그대로 머문다. 같은 지역의 다른 포인트에서 이미 통과했다면(미션 완료 전이라도) 재시도는 서버에서 차단되고 "그 포인트로 돌아가라"는 안내만 표시.
- AI 판정은 CLIP/CNN 임베딩이 아니라 **VQA(자연어 질문) 방식**, **OpenAI 비전 지원 모델**(GPT-4o 계열, 정확한 모델명은 env var로 설정) 사용 (2026-07-14 변경: 원래 Claude 비전 모델이었으나 결제 방식 이유로 OpenAI로 전환).
- 지도는 Kakao Maps JS SDK(무료 쿼터로 충분) + 브라우저 GPS로 내 위치 표시. GPS는 내비게이션 편의용이며 판정에는 쓰지 않음.
- 장소 등록은 현장(답사 당일) 빠른 입력이 기본 전제 — "내 위치 사용" GPS 버튼 + 폰 카메라 촬영이 메인 흐름, judgePrompt는 공통 기본 템플릿에서 시작해 나중에 원격으로 다듬음.
- 재시도 횟수 제한 없음. "임원 도움 요청" 버튼 상시 노출 — 사전 답사가 사실상 불가능해 AI 판정을 미리 검증할 기회가 거의 없으므로 안전망이 아니라 주력 백업 수단.
- 임원 인증 없음 — 추측 어려운 비밀 URL 경로로만 관리자 화면 접근 (로그인/비밀번호 없음).
- 기술 스택: Next.js(App Router, TS) + Prisma + Supabase(Postgres+Storage, 로컬은 SQLite로 시작 가능) + Kakao Maps + OpenAI vision API + Vercel 배포.

## 디자인

방향 재확정 (2026-07-15, notes/26.07.15.md 참고). 실제 지도를 화면에 넣어보고 나서 "장난감 같다"는 피드백을 거쳐, 베이스 톤은 유지하되 표현 방식을 바꿨다.

- **베이스**: 밝은 배경 + 짙은 잉크 텍스트 + 오렌지 액센트라는 큰 방향은 그대로, 다만 카키톤 하나가 아니라 **투톤(카키 페이지 + 흰색 계기 카드)** — 헤더/여백은 카키(`--color-paper` `#F5F3EC`), 지도+패널을 감싸는 프레임 내부는 거의 흰색(`--color-paper-panel` `#FFFFFF`)에 가깝게. 텍스트 `--color-ink` `#1C211D`, 액센트 `--color-accent` `#E1591C`. 팀(A~D) 컬러는 배경과 구분되는 별도 4색 사용.
- **테두리/모서리는 두꺼운 flat 잉크선 대신 헤어라인(1px) + 미세한 그림자**로. 알약형(rounded-full) 배지·버튼도 각진 카드(6~8px 라운드)로 교체 — 원인이 색이 아니라 이 두꺼운 테두리와 둥근 알약형이었음.
- **미션 통과 리빌 화면**: 원형 잉크 스탬프 대신 보딩패스/permit 카드 톤의 각진 카드(PASS + 코드 + 미션 내용)로 변경, 뜨는 순간 오렌지 빛이 1초 정도 한 번 번지는 정도로만 임팩트를 준다 (반복 재생 안 되게 주의).
- **"Borderless" 모티프**: 헤더에 지역 진행 스테퍼(a-b-c-d, 완료/현재/예정 상태 표시) + 현재 위치를 나타내는 점을 두고, 그 점에서 짧은 연결선이 지도 프레임 위 목적지 배지로 이어지게 해서 "경계를 넘어 계속 나아간다"는 걸 장식이 아니라 정보 구조로 표현한다.
- 상시 노출되는 "임원 도움 요청" 버튼은 어떤 화면에서도 눈에 띄게 유지 — 단, 다른 UI(특히 패널 텍스트)를 가리지 않도록 여유 공간을 확보해야 함.
- **차단/대기 안내(LOCKED/CLOSED)**: "차례가 아님"/"마감" 같은 상태는 평문으로 두면 잘 안 읽혀서, PASS 카드와 같은 카드 골격(헤어라인 accent 테두리 + 점선 헤더 구분선)에 영문 스탬프 라벨(LOCKED/CLOSED)을 얹어 표시한다. 이모지·흔들림 애니메이션·진동은 시도했다가 "AI가 만든 티가 난다"는 피드백으로 뺐고, 카드가 한 번 살짝 떠오르는 정도의 절제된 모션만 남겼다. 문구도 "~해요" 대신 표지판처럼 짧게 끊는 톤("A지역부터 먼저 완료").
- **PC 전용 레이아웃**: 참가자 지도 화면은 `lg:` 이상에서만 지도(좌, ~60%)·상세 패널(우, ~40%) 2단으로 나뉘고 그 아래 폭에서는 기존 모바일 세로 스택 그대로다. 카카오 지도는 컨테이너 폭이 바뀌어도 스스로 다시 그리지 않아 `ResizeObserver` + `relayout()`으로 보정한다.
- 비교 검토했던 다른 방향(워십 슬라이드형 다크 톤, 여권/스탬프북형 크래프트지, 팀 스코어보드형 경쟁 톤, 다크 나이트옵스)은 기각 — 다크 배경은 실외 가독성이 떨어지고, 스코어보드 톤은 말씀/기도 리빌과 어울리지 않는다고 판단.
- 정확한 토큰 값은 `src/app/globals.css`의 `@theme` 블록이 최신 소스.

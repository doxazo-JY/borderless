# 데일리 노트 — 2026.07.14

## 오늘 요약 (공유용)

Borderless 프로젝트를 실제 코드로 시작한 첫날. 기획 단계에서 정해둔 내용(4팀×2조, 4지역×4포인트, 지역 순서 강제, 캡 선착순, VQA 판정)을 바탕으로 기술 스택을 확정하고(Next.js + Prisma + Supabase + Kakao Maps + OpenAI), 참가자 화면의 첫 두 단계 — 팀/조 선택과 지도 화면 — 을 실제로 동작하게 만들었다. 디자인 방향도 이날 확정했다 (오리엔티어링 컨트롤 베이스 + 리빌 화면만 스탬프 연출). 다만 카카오맵 마커 클릭이 끝내 안 먹혀서, 그 문제는 다음날(7/15)로 넘어갔다.

## 오늘 한 일

1. Next.js(App Router, TS) 스캐폴딩, Prisma 스키마 초안 + 로컬 SQLite로 개발 시작
2. `CLAUDE.md` / 기획서 문서 정리 (스캐폴딩 중 템플릿에 덮어써진 걸 복구)
3. Supabase Postgres로 전환 (Prisma 7의 `prisma.config.ts` 기반 설정으로 이전)
4. 팀/조 선택 화면 + 쿠키 기반 그룹 세션(로그인 없이 목록 선택)
5. Kakao Maps JS SDK 연동 — 지도/내 위치는 뜨는데, 마커 클릭 판정은 이날 안에 못 고치고 넘어감
6. 디자인 방향 확정: 오리엔티어링 컨트롤(밝은 종이/카키톤 + 잉크 텍스트 + 오렌지 액센트), 리빌 화면만 원형 스탬프 예외

## 이슈 & 해결

| 이슈 | 원인 | 해결 |
|---|---|---|
| `CLAUDE.md`가 사라지고 템플릿 내용으로 바뀜 | `create-next-app` 스캐폴딩 중 자체 `CLAUDE.md`(AGENTS.md 안내용)가 프로젝트 파일에 덮어써짐 | 기획 대화 내용을 바탕으로 원래 내용 복구 |
| Prisma 스키마 검증 에러 | Prisma 7부터 `datasource` 블록에 `url`/`directUrl`을 인라인으로 못 씀 | `prisma.config.ts`로 이전, migrate는 `DIRECT_URL`, 런타임은 `DATABASE_URL`(풀링) 사용 |
| `PrismaClient`에서 "password must be a string" 에러 | 임시 테스트 스크립트에서 정적 import가 `dotenv.config()` 호출보다 먼저 평가됨(ES 모듈 호이스팅) | 테스트 스크립트에선 공용 싱글턴 대신 그 자리에서 직접 클라이언트 생성 |
| Supabase 인증 실패(P1000) | 비밀번호 재설정 직후 전파 지연 (인코딩 문제 아니었음) | 잠시 대기 후 재시도로 해결 |
| Kakao Maps "OPEN_MAP_AND_LOCAL service disabled" | 카카오맵 제품 자체가 콘솔에서 활성화 안 돼 있었음(도메인 등록과 별개) | 카카오 개발자 콘솔 → 제품 설정 → 카카오맵 활성화 |
| 카카오 지도 중복 생성 (React StrictMode) | 개발 모드에서 effect가 두 번 실행됨 | `mapCreatedRef`로 중복 생성 가드 |
| 지도 컨테이너 높이가 0 | `body`에 `min-h-full`을 썼는데 `height`가 아니라 `min-height`라 자식 요소의 flex-grow 높이 해석이 깨짐 | `h-full`로 수정 |
| 푸시할 때마다 계정 선택 창이 뜸 | Windows 자격 증명 관리자에 GitHub 계정 여러 개 저장돼 있음 | remote URL에 사용자명을 박아서 특정 계정으로 고정 |

지도 마커 클릭이 이날 안에 안 풀려서(→ 7/15 노트로 이어짐), 오늘은 지도가 뜨는 것까지만 확인하고 마무리.

## 느낀 점 / 메모

- 카카오맵처럼 오래된 서드파티 SDK는 "공식 문서대로 하면 되겠지"가 잘 안 통했다 — 실제로 이벤트를 찍어보고 DOM을 뜯어보며 원인을 좁혀가는 과정이 필요했다.
- Prisma 7이 예전 버전과 설정 방식이 꽤 달라서(`prisma.config.ts`), 훈련 데이터 기준으로 짐작하지 말고 실제 에러 메시지를 보고 새로 확인하는 게 나았다.
